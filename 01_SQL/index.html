BDD

Les bases de données sont des magasins où l'on peut stocker des informations et permettent notamment d'effectuer différentes opérations de lecture et d'écriture

Elles sont efficaces pour retrouver des informations qui y sont stockées. Vous pouvez interroger une base de données, par exemple : "Combien d'utilisateurs sont inscrits sur mon site web ?" ou "Quel article dans mon blog contient tel ou tel mot-clé ?".

La force des bases de données réside également dans leur possibilité de lier les différentes informations qui y sont préalablement stockées.

SGBD

SGBD (Système de Gestion de Base de Données) permet d'accueillir, d'exploiter et de faire fonctionner les bases de données (moteur). Nous utiliserons MySQL. D'autres SGBD existent tels que : Oracle, Sql Server, PostgreSQL, etc...

MySQL

MySQL est un SGBDR (Système de Gestion de Base de Données Relationnelle) C'est l'un des systèmes les plus populaires au monde et est très utilisé dans le monde du web.

Comme son nom l'indique, MySQL utilise le langage SQL pour effectuer des requêtes.

Toutes ces actions de gestion et manipulation passeront par une requête SQL.

SQL

SQL (Structured Query Language). C'est un langage de requête structuré. Il s'agit d'un langage informatique permettant d'intéragir avec des bases de données relationnelles

Une fois que les informations ont été enregistrées, il est important de pouvoir les gérer :

  • ajout
  • modification
  • suppression
  • consultation

phpMyAdmin

phpMyAdmin (souvent abrégé PMA) est une application web de gestion pour MySQL. Il est écrit en PHP et permet de gérer les BDD dans une interface visuelle simple et intuitive.

Il est important de noter que phpMyAdmin n'est qu'une application. Il est tout à fait possible de gérer MySQL avec d'autres applications similaire ou en ligne de commande

CRUD

CRUD (Create, Read, Update, Delete) désigne les 4 opérations de base pour le stockage et la manipulation d'informations dans une base de données.

  • Create : Inserer des nouveaux enregistrements (écriture)
  • Read : Récupérer des enregistrements (lecture)
  • Update : Mettre à jour des enregistrements (écriture)
  • Delete : Supprimer des enregistrements (écriture)

Les opérations correspondent chacune à un type de requêtes SQL
